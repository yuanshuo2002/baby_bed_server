## 关系说明

| 关系 | 说明 |
|------|------|
| **User ↔ Family** | 通过 `family_members` 关联表实现多对多。一个用户可以加入多个家庭，一个家庭可以有多个成员 |
| **Family → Baby** | 一对多。一个家庭可以管理多个宝宝 |
| **Baby ↔ Device** | 多对一。一个设备只能绑定一个宝宝，一个宝宝可以绑定多个设备 |
| **Device → SensorDataRaw** | 一对多。设备上报的传感器数据都关联到该设备 |
| **SensorDataRaw → Baby** | 多对一。每条传感器数据关联一个宝宝（直接关联） |
| **Device → Video** | 一对多。设备上传的视频通过 device_sn 关联 |
| **Baby → BabyStatusLog** | 一对多。宝宝的状态变化日志 |
| **Baby → CryEvent** | 一对多。哭闹事件记录 |
| **Baby → DangerEvent** | 一对多。危险事件记录 |

## 业务流程

### 1. 用户注册/登录
```
用户通过手机号或微信登录 → 创建/查找 User 记录
```

### 2. 创建/加入家庭
```
用户创建家庭 → 创建 Family + 当前用户作为 FamilyMember
用户通过 family_code 加入已有家庭 → 创建 FamilyMember
```

### 3. 添加宝宝
```
家庭成员添加宝宝 → 创建 Baby（关联当前 Family）
```

### 4. 设备注册与绑定
```
硬件设备注册 → 创建设备记录（baby_id 为空）
小程序绑定设备 → 更新 Device.baby_id 指向目标宝宝
```

### 5. 数据上报流程

**传感器数据上报**:
```
设备 → sensor_data_raw（device_sn + baby_id）
      → 更新设备在线状态
```

**视频上传**:
```
设备 → videos（只有 device_sn）
     → 通过 Device.baby_id 获取关联的宝宝
```

### 6. 状态检测与事件生成
```
传感器数据 → 状态分类（sleeping/awake/playing/crying/danger）
          → baby_status_log（状态变化时记录）
          → cry_event（哭闹时）
          → danger_event（危险时）
```

## 关联数据清理

删除设备/宝宝/家庭时，需要按以下顺序清理关联数据：

### 删除设备（device_sn）
```sql
-- 1. 清理视频记录
DELETE FROM videos WHERE device_sn = 'xxx';

-- 2. 清理传感器数据
DELETE FROM sensor_data_raw WHERE device_sn = 'xxx';

-- 3. 清理状态日志（通过 device_sn）
DELETE FROM baby_status_log WHERE device_sn = 'xxx';

-- 4. 解绑设备（设置 baby_id 为空）
UPDATE devices SET baby_id = NULL WHERE device_sn = 'xxx';
-- 或删除设备
DELETE FROM devices WHERE device_sn = 'xxx';
```

### 删除宝宝（baby_id）
```sql
-- 1. 解绑设备（设置 baby_id 为空）
UPDATE devices SET baby_id = NULL WHERE baby_id = xxx;

-- 2. 清理哭闹事件
DELETE FROM cry_event WHERE baby_id = xxx;

-- 3. 清理危险事件
DELETE FROM danger_event WHERE baby_id = xxx;

-- 4. 清理状态日志
DELETE FROM baby_status_log WHERE baby_id = xxx;

-- 5. 清理传感器数据
DELETE FROM sensor_data_raw WHERE baby_id = xxx;

-- 6. 删除宝宝记录
DELETE FROM babies WHERE id = xxx;
```

### 删除家庭（family_id）
```sql
-- 1. 获取家庭下所有宝宝
SELECT id FROM babies WHERE family_id = xxx;

-- 2. 清理宝宝及关联数据（参考"删除宝宝"步骤）

-- 3. 清理家庭成员
DELETE FROM family_members WHERE family_id = xxx;

-- 4. 删除家庭
DELETE FROM families WHERE id = xxx;
```

### 删除用户（user_id）
```sql
-- 1. 离开所有家庭（删除家庭成员关系）
DELETE FROM family_members WHERE user_id = xxx;

-- 2. 删除用户（可选，取决于业务需求）
DELETE FROM users WHERE id = xxx;
```
